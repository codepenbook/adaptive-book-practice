Добавляем адаптив для "Header"

Важное замечание!

Обратите внимание на структуру вложенности элементов блоков. Их правильное построение играет важную роль в адаптивной верстке сайта.



Или вот, один из сложных примеров, он содержит в себе много вложенности.



Конечно вы можете делать это по-другому, однако есть определенные правила, которых стоит придерживаться. Такой подход делает вашу верству удобной и профессиональной!


Более подробно с этим можете ознакомиться по ссылке:

Методология БЭМ - Быстрый старт

Теперь перейдем непосредственно к добавлению адаптивности в header.


Для удобства закомментируйте всё, кроме header и секции hero.


Попробуйте уменьшить разрешение экрана и станет видно, что ниже 800px, header выходит за контейнер.



Исправим это следующим образом:


@media (max-width: 820px) {

/*На ширине экрана 820px, уменьшим правые отступы для всех menu__item, кроме последнего. */

.menu__list>*:not(:last-child) {

margin: 0px 20px 0px 0px;

}

}



Теперь header отображается адекватно и на ширине 768px, можно навигацию закинуть в меню бургер, однако у нас учебник по адаптивной верстки и пока на этом разрешении, для header__menu установим display: none.

Правильное адаптивное меню бургер на HTML CSS и jQuery.

На этом с header закончим, т.к. до 320px, всё отлично смотриться.



Добавляем адаптив для "Hero"

При уменьшении экрана на 768px, отступы сверху и снизу у hero__body, слишком большие, поэтому уменьшим их.


@media (max-width: 768px) {

.hero__body {

padding: 150px 0px 90px 0px;

}

}



Далее для ширины в 500px произведем следующие действия (изменения увидите сами):


@media (max-width: 500px) {

.hero__body {

padding: 110px 0px 40px 0px;

/* Блок, который содержит в себе заголовок, текст и кнопки, будет занимать теперь 100% ширины */

width: 100%;

/* Весь текст в нем будет по центру */

text-align: center;

}

.hero__title {

font-size: 36px;

line-height: 40px;

}

.hero__desc {

margin: 0px 0px 20px 0px;

font-size: 16px;

line-height: 25px;

}

.hero__buttons {

/* Кнопки теперь не вряд, а в колонку */

flex-direction: column;

}

.hero__buttons>*:not(:last-child) {

margin: 0px 0px 10px 0px;

}

}


С секцией hero на этом тоже можно закончить.



Добавляем адаптив для "Easy Useful"

Для этой секции, вы не поверите, достаточно всего три строчки кода:


@media (max-width: 768px) {

/* header-block это БЭМ блок, он используется в этой секции и секции services, поэтому нам достаточно один раз уменьшить отступ сверху и снизу. */

.header-block {

padding: 50px 0px 50px 0px;

}

.easy-useful__row {

/* Теперь скажем так, наши карточки не ряд, а в колонку */

flex-direction: column;

/* Уменьшим отступ снизу до кнопки */

margin: 0px 0 40px 0;

}

}


На этом с этой секцией всё!



Добавляем адаптив для "Count"

И здесь аналогично всего три строчки кода:


@media (max-width: 820px) {

/* Дадим этим блокам возможность переноситься ниже */

.count__row {

flex-wrap: wrap;

}

.count__column {

/* Укажем, что на этой ширине экрана, блоки будут не в ряд по 4, а по 2 */

flex: 0 1 50%;

}

}


@media (max-width: 500px) {

.count__column {

/* Укажем, что на этой ширине экрана, блоки будут не в ряд по 2, а по 1 */

flex: 0 1 100%;

}

}


Вот и всё, круто, правда? Во всём помогает правильная структура блоков по методологии БЭМ!



Добавляем адаптив для "Services"

Всего две строчки...

Добавляем адаптив для "Testimonials"

Как говорилось ранее, это сложная секция, но даже она при правильной структуре блоков, оказывается не трудной.


@media (max-width: 860px) {

.testimonials__body {

/* Уменьшим отступ сверху и снизу */

padding: 40px 0;

}

.testimonials__row {

/* Для двух больших блоков: с заголовоком, с текстом, кнопкой и комментарием пользователей укажем, чтобы они былы в колонку */

flex-direction: column;

}

}

.testimonials__row>*:not(:last-child) {

/* Уменьшим между ними отступ */

margin: 0px 0px 40px 0px;

}



@media (max-width: 500px) {

.testimonials__column {

/* Теперь эти два больших блока, занимают всю ширину */

width: 100%;

}

/* Для элементов внутри этих двух блоков укажем, чтобы они были в колонку, по середине и текст тоже был по середине */

.testimonials__item {

display: flex;

flex-direction: column;

align-items: center;

text-align: center;

}

/* В первом блоке для заголовка уменьшим размер шрифта и междустрочный отступ*/

.item-testimonials__title {

font-size: 36px;

line-height: 40px;

}

.item-testimonials__text {

/* В первом блоке для текста уменьшим отступ до кнопки */

margin: 0px 0px 20px 0px;

}

.item-testimonials__row>*:not(:last-child) {

/* Уменьшим отступ между этими двумя блоками */

margin: 0px 0px 40px 0px;

}

.comment__item {

/* Для элементов комментария укажем, чтобы они были в колонку */

flex-direction: column;

}

.comment__item>*:not(:last-child) {

/* И уберем отступы */

margin: 0;

}

}





Добавляем адаптив для "Footer"

С footer тоже все просто:


@media (max-width: 768px) {

/*Уменьшим отступ снизу и сверху, для двух блоков footer укажем, чтобы они были в колонку и по середине. */

.footer__body {

padding: 40px 0;

display: flex;

flex-direction: column;

align-items: center;

}

.footer__up,

.footer__bottom {

/* Укажем, чтобы элементы внутри footer__up и footer__bottom были в колонку. */

flex-direction: column;

}

.footer__up>*:not(:last-child),

.footer__bottom>*:not(:last-child) {

/* Укажем, чтобы элементы внутри footer__up и footer__bottom имели нижний отступ, кроме последнего элемента */

margin: 0px 0px 40px 0px;

}

.footer__menu .menu__list {

/* Немного приукрасим навигацию в footer, чтобы она была в колонку */

flex-direction: column;

}

.footer__menu .menu__list>*:not(:last-child) {

/* И уберем отступы у элементов навигации footer */

margin: 0;

}

}



Больше изменять ничего не нужно, на ширине 320px, footer выглядит хорошо.
